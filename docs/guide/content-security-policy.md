# Content Security Policy (CSP) and Nonces

Autumn includes automatic Content-Security-Policy hardening as part of its built-in security headers middleware. By default, Autumn enforces a strict, secure policy that blocks inline scripts and styles to prevent Cross-Site Scripting (XSS) attacks.

When inline scripts or styles are required, Autumn provides **auto-generated per-request CSP nonces** so that you can adopt a strict CSP without resorting to the insecure `'unsafe-inline'` rule.

---

## Acceptance Criteria & Nonce Flow

On every incoming request:
1. If CSP is enabled, a fresh cryptographically random nonce (>=128 bits, base64) is generated by the `SecurityHeaders` middleware.
2. The nonce is attached to the request extensions.
3. Handlers and templates can extract the nonce using the `CspNonce` extractor.
4. The middleware automatically rewrites the `Content-Security-Policy` header in the response to include `'nonce-<value>'` on the configured directives (default: `script-src` and `style-src`).

---

## Configuration

The CSP and nonce rewriting feature is configured in `autumn.toml` under `[security.headers]`:

```toml
[security.headers]
# Enable standard Content-Security-Policy (defaults to strict same-origin)
content_security_policy = "default-src 'self'; img-src 'self' data:; style-src 'self'; script-src 'self'; connect-src 'self'; form-action 'self'; frame-ancestors 'none'; base-uri 'self'"

[security.headers.csp_nonce]
# Toggle the per-request nonce generation feature. Default: true.
enabled = true

# Select which directives receive the nonce. Default: ["script-src", "style-src"]
directives = ["script-src", "style-src"]
```

### Backwards Compatibility & Opt-Out

If you override `content_security_policy` to a custom literal string in `autumn.toml` (or via environment variables) and do not explicitly configure `csp_nonce.enabled = true`, the framework will **automatically opt out** of nonce injection to avoid surprising or rewriting your explicit policies.

To disable both the CSP header and nonce generation entirely (not recommended for production):
```toml
[security.headers]
content_security_policy = ""
```

## Worked Example

### 1. Extracting `CspNonce` in Route Handlers

To retrieve the per-request nonce in your handler, add the `CspNonce` parameter. It is re-exported from `autumn_web::prelude` and can be wrapped in an `Option` if CSP could be disabled:

```rust
use autumn_web::prelude::*;

#[get("/")]
pub async fn index(nonce: Option<CspNonce>) -> Markup {
    html! {
        (layout("My Page", nonce.as_ref(), html! {
            h1 { "Welcome!" }
        }))
    }
}
```

### 2. Using the Nonce in Maud Templates

Maud templates can conditionally apply the nonce to `<script>` and `<style>` tags using Maud's optional attribute syntax:

```rust
use autumn_web::prelude::*;

pub fn layout(title: &str, nonce: Option<&CspNonce>, content: Markup) -> Markup {
    html! {
        (PreEscaped("<!DOCTYPE html>"))
        html {
            head {
                title { (title) }
                
                // 1. Same-origin script (no nonce required by 'self')
                script src="/static/js/app.js" {}

                // 2. Nonce-protected inline script block
                script nonce=[nonce.map(|n| n.nonce())] {
                    "console.log('App initialized with CSP nonce:', '" (nonce.map(|n| n.nonce()).unwrap_or_default()) "');"
                }

                // 3. Nonce-protected inline style block
                style nonce=[nonce.map(|n| n.nonce())] {
                    "body { background-color: #fafafa; }"
                }

                // 4. Third-party widget script
                // Note: You must also add the third-party domain to your script-src directive
                script nonce=[nonce.map(|n| n.nonce())] src="https://static.cloudflareinsights.com/beacon.min.js" {}
            }
            body {
                (content)
            }
        }
    }
}
```
