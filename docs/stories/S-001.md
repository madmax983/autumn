I want a **properly structured Cargo workspace with separate crates for runtime and proc macros**
So that **all subsequent development has a solid foundation with correct dependency boundaries**

---

## Description

### Background
Autumn is organized as a Cargo workspace with three crates: `autumn` (runtime + re-exports), `autumn-macros` (proc macros), and `autumn-cli` (scaffolding). Rust requires proc macros to live in a separate crate with `proc-macro = true`. The workspace structure determines the dependency graph, build performance, and developer ergonomics for the entire project.

This is the absolute foundation — every other story depends on this structure existing and compiling.

### Scope
**In scope:**
- Workspace root `Cargo.toml` with members
- `autumn` crate skeleton (lib.rs with placeholder re-exports)
- `autumn-macros` crate skeleton (lib.rs with `proc-macro = true`)
- Core dependencies declared in workspace `[workspace.dependencies]`
- `autumn` depends on `autumn-macros` (path dependency)
- Both crates compile on stable Rust with `cargo build`
- Workspace-wide lint configuration (clippy pedantic/nursery)
- Edition 2024

**Out of scope:**
- `autumn-cli` crate (Sprint 9, S-034)
- Any actual functionality — this is skeleton only
- Examples directory structure
- CI pipeline (S-043)

### Expected Result

After this story, `cargo build` from the workspace root succeeds. `cargo test` runs (even if there are zero tests). `cargo clippy` passes. The crate structure matches the architecture document.

## Technical Notes

### Workspace Layout

```
autumn/
├── Cargo.toml                    # Workspace root
├── autumn/                       # Main crate (runtime + re-exports)
│   ├── Cargo.toml
│   └── src/
│       └── lib.rs
└── autumn-macros/                # Proc macro crate
    ├── Cargo.toml
    └── src/
        └── lib.rs
```

### Dependency Graph

```
autumn ─────────► autumn-macros
   │
   ├──► axum (workspace dep)
   ├──► tokio (workspace dep)
   ├──► serde (workspace dep)
   └──► ...

autumn-macros ──► syn (workspace dep)
               ► quote (workspace dep)
               ► proc-macro2 (workspace dep)
```

### Key Architecture Decisions

- **`autumn-macros` must NOT depend on any runtime crate** (`axum`, `tokio`, `diesel`, etc.). It generates code that *references* `autumn_web::` paths, but doesn't import them. This is a hard boundary from the architecture document.
- **`autumn` re-exports `autumn-macros`** so users only need `autumn` in their `Cargo.toml`.
- **Workspace-level dependency management** ensures version consistency across crates.
- **Edition 2024** per CLAUDE.md coding standards.

### Workspace Cargo.toml Sketch

```toml
[workspace]
members = ["autumn", "autumn-macros"]
resolver = "3"

[workspace.package]
edition = "2024"
license = "MIT OR Apache-2.0"
repository = "https://github.com/markm/autumn"

[workspace.dependencies]
# Runtime
axum = "0.8"
tokio = { version = "1", features = ["full"] }
serde = { version = "1", features = ["derive"] }

# Proc macro
syn = { version = "2", features = ["full"] }
quote = "1"
proc-macro2 = "1"
```

### Lint Configuration

```toml
[workspace.lints.clippy]
pedantic = { level = "warn", priority = -1 }
nursery = { level = "warn", priority = -1 }
```

### What "Placeholder" Means

`autumn-macros/src/lib.rs` should have a minimal compilable proc macro (can be a no-op that just returns the input TokenStream). `autumn/src/lib.rs` should re-export the macro crate. The goal is structural correctness, not functionality.
