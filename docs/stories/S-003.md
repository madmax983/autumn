## User Story

As a **framework user**
I want to **annotate async functions with `#[post]`, `#[put]`, and `#[delete]` to define route handlers for those HTTP methods**
So that **I can build full CRUD APIs with the same minimal-boilerplate experience as `#[get]`**

---

## Description

### Background
With `#[get]` implemented in S-002, adding support for the remaining HTTP methods is straightforward. Each macro is a one-line wrapper that calls the existing `route_macro()` function in `autumn-macros/src/route.rs` with a different HTTP method and Axum routing function name. The generated code is identical in shape to `#[get]` — the only difference is the `http::Method` variant and the `axum::routing::*` function used.

### Scope
**In scope:**
- `#[post("/path")]` proc macro attribute
- `#[put("/path")]` proc macro attribute
- `#[delete("/path")]` proc macro attribute
- Re-exports from `autumn` crate (`pub use autumn_macros::{post, put, delete}`)
- Tests verifying each method generates the correct `http::Method`

**Out of scope:**
- Request body extraction (works automatically since the macro doesn't touch the function signature)
- `#[patch]`, `#[head]`, `#[options]` (can be added later with the same pattern)
- Path parameter extraction (S-006)

## Technical Notes

### Implementation

Each macro is a thin wrapper in `autumn-macros/src/lib.rs` calling the shared `route_macro()` core:

```rust
#[proc_macro_attribute]
pub fn post(attr: TokenStream, item: TokenStream) -> TokenStream {
    route::route_macro("POST", "post", attr.into(), item.into()).into()
}

#[proc_macro_attribute]
pub fn put(attr: TokenStream, item: TokenStream) -> TokenStream {
    route::route_macro("PUT", "put", attr.into(), item.into()).into()
}

#[proc_macro_attribute]
pub fn delete(attr: TokenStream, item: TokenStream) -> TokenStream {
    route::route_macro("DELETE", "delete", attr.into(), item.into()).into()
}
```

The `route_macro()` function (implemented in S-002) takes the HTTP method as an uppercase string for the `http::Method` constant (`GET`, `POST`, etc.) and the lowercase Axum routing function name (`get`, `post`, etc.). No changes to `route.rs` were required.

### Re-exports

Added to `autumn/src/lib.rs`:

```rust
pub use autumn_macros::{delete, get, post, put, routes};
```

### Testing Strategy

Tests in `autumn/tests/route_macro.rs` verify each method macro:
- `post_route_info_has_correct_method` — asserts `http::Method::POST`
- `post_route_info_has_correct_path` — asserts path matches attribute
- `post_route_info_has_correct_name` — asserts function name
- `put_route_info_has_correct_method` — asserts `http::Method::PUT`
- `put_route_info_has_correct_path` — asserts path matches attribute
- `put_route_info_has_correct_name` — asserts function name
- `delete_route_info_has_correct_method` — asserts `http::Method::DELETE`
- `delete_route_info_has_correct_path` — asserts path matches attribute
- `delete_route_info_has_correct_name` — asserts function name

All compile-fail tests from S-002 (non-async, empty path, non-function) apply identically since they share the same `route_macro()` core.

### Dependencies Added

None — all dependencies were already in place from S-002.
