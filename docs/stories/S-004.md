## User Story

As a **framework user**
I want **`#[axum::debug_handler]` to be automatically applied to my route handlers in debug builds**
So that **I get clear, actionable error messages when my handler signatures are wrong, without any manual annotation or runtime cost in production**

---

## Description

### Background
Axum's `#[debug_handler]` attribute dramatically improves error messages for handler functions with incorrect signatures (wrong extractor types, missing trait implementations, etc.). Without it, Axum handler errors produce opaque trait-bound failures that are notoriously hard to debug.

The `debug_handler` was already implemented as part of S-002 via `cfg_attr(debug_assertions, ...)` in the `route_macro()` code generation. This story covers verification and test coverage — ensuring that different return types compile correctly with `debug_handler` active, and that release builds don't include it.

### Scope
**In scope:**
- Verification that `debug_handler` is applied in debug builds (`cfg(debug_assertions)`)
- Verification that `debug_handler` is NOT applied in release builds
- Test coverage for handlers with various return types under `debug_handler`:
  - `&'static str`
  - `String`
  - `axum::Json<serde_json::Value>`
  - `http::StatusCode`
- Confirming zero cost in release builds

**Out of scope:**
- Changing the `debug_handler` implementation (already done in S-002)
- Custom error messages for handler signature mistakes (S-007)
- Testing with extractors (comes with S-006)

## Technical Notes

### Implementation (from S-002)

The `route_macro()` function in `autumn-macros/src/route.rs` generates:

```rust
#[cfg_attr(
    debug_assertions,
    ::autumn_web::reexports::axum::debug_handler(state = ::autumn_web::AppState)
)]
#input_fn
```

This uses `cfg_attr` so the attribute is only applied when `debug_assertions` is enabled (the default for `cargo build` and `cargo test`). Release builds (`cargo build --release`) strip it entirely.

The `state = ::autumn_web::AppState` parameter is required because `debug_handler` needs to know the state type to validate extractor bounds. The placeholder `AppState` struct (defined in `autumn/src/lib.rs`) satisfies this requirement.

### Testing Strategy

Tests in `autumn/tests/route_macro.rs` verify different return types compile under `debug_handler`:

- `string_return_type_compiles_with_debug_handler` — handler returning `String`
- `json_return_type_compiles_with_debug_handler` — handler returning `axum::Json<serde_json::Value>`
- `status_code_return_type_compiles_with_debug_handler` — handler returning `http::StatusCode`

The test strategy is "compile-pass": if the test file compiles, `debug_handler` accepts the return type. The tests also verify the route info fields are correct, providing both compile-time and runtime validation.

Release build verification can be done manually with `cargo expand --release` to confirm `debug_handler` is absent.

### Dependencies Added

**`autumn/Cargo.toml` (dev-dependencies):**
- `serde_json = "1"` (for `axum::Json<serde_json::Value>` return type in tests)
