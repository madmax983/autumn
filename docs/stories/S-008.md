## User Story

As a **framework user**
I want to **write `#[autumn_web::main]` on my main function**
So that **the Tokio async runtime is set up without me configuring it manually**

---

## Description

### Background
`#[autumn_web::main]` is a thin wrapper around `#[tokio::main]`. In v0.1, it just sets up the runtime. Future versions may add custom panic handlers, runtime tuning, or instrumentation. The real work happens in `autumn_web::app().run()`, not in this macro.

The architecture doc is explicit: keep this macro minimal. The runtime configuration belongs in the app builder, not the entry point macro.

### Scope
**In scope:**
- `#[autumn_web::main]` proc macro attribute in `autumn-macros`
- Expansion to `#[tokio::main]`
- Validation: must be `async fn main()`
- Re-export so users write `#[autumn_web::main]`
- Compile-pass and compile-fail tests

**Out of scope:**
- Config loading (happens in `app().run()`, S-009)
- Database pool creation (S-016)
- Custom runtime configuration (future)

## Technical Notes

### Implementation

Create `autumn-macros/src/main_macro.rs`:

```rust
use proc_macro2::TokenStream;
use quote::quote;
use syn::ItemFn;

pub fn main_macro(item: TokenStream) -> TokenStream {
    let input_fn: ItemFn = match syn::parse2(item) {
        Ok(f) => f,
        Err(err) => return err.to_compile_error(),
    };

    if input_fn.sig.asyncness.is_none() {
        return syn::Error::new_spanned(
            input_fn.sig.fn_token,
            "the main function must be async",
        )
        .to_compile_error();
    }

    quote! {
        #[::autumn_web::reexports::tokio::main]
        #input_fn
    }
}
```

### Entry Point

```rust
// autumn-macros/src/lib.rs
mod main_macro;

#[proc_macro_attribute]
pub fn main(_attr: TokenStream, item: TokenStream) -> TokenStream {
    main_macro::main_macro(item.into()).into()
}
```

### Re-exports

`tokio` needs to be in `autumn_web::reexports`:
```rust
pub mod reexports {
    pub use axum;
    pub use http;
    pub use tokio;
}
```

The `main` attribute macro is re-exported as `autumn_web::main` automatically since `autumn` re-exports `autumn_macros`.

### Testing

Compile-pass test (in an integration test that has a `main` function):
```rust
// Can't easily test #[autumn_web::main] in a #[test] since it wraps main().
// Use trybuild compile-pass test instead.
```

Compile-fail test:
```rust
// tests/compile-fail/non_async_main.rs
#[autumn_web::main]
fn main() {
    println!("not async");
}
```
