## User Story

As a **framework user**
I want to **write `autumn::app().routes(my_routes).run().await` and have a running HTTP server**
So that **I can go from annotated handlers to a listening server with one line of bootstrap**

---

## Description

### Background
This is where Autumn becomes a framework. The app builder collects routes, loads config, builds an Axum router, and starts a server with graceful shutdown. After this story, Autumn can serve HTTP responses.

For v0.1/Sprint 3, the builder is simplified — no database pool (S-016), no logging middleware (S-028), no request ID (S-011). Just: config → routes → router → server.

### Scope
**In scope:**
- `autumn::app()` function returning `AppBuilder`
- `AppBuilder::routes(Vec<Route>)` — chainable, collects routes
- `AppBuilder::run()` — loads config, builds Axum router from routes, binds to address, starts server
- Empty routes panic with clear message
- Startup log of mounted routes (method + path)
- Graceful shutdown on Ctrl+C
- Server binds to `config.server.host:config.server.port`

**Out of scope:**
- Database pool creation (S-016)
- Custom middleware / layers (v1.0)
- `AppBuilder::merge()` for raw Axum routers (v1.0)
- Health check endpoint (S-029)
- Request ID middleware (S-011)
- Structured logging (S-028) — use `println!` for now

### The "It's Alive" Test

After this story, this program works:
```rust
use autumn::{get, routes};

#[get("/hello")]
async fn hello() -> &'static str {
    "Hello, Autumn!"
}

#[autumn::main]
async fn main() {
    autumn::app()
        .routes(routes![hello])
        .run()
        .await;
}
```

`cargo run` → visit `http://localhost:3000/hello` → see "Hello, Autumn!"

## Technical Notes

### AppBuilder (simplified for Sprint 3)

```rust
// autumn/src/app.rs

use crate::config::AutumnConfig;
use crate::route::Route;
use crate::AppState;

pub fn app() -> AppBuilder {
    AppBuilder {
        routes: Vec::new(),
    }
}

pub struct AppBuilder {
    routes: Vec<Route>,
}

impl AppBuilder {
    pub fn routes(mut self, routes: Vec<Route>) -> Self {
        self.routes.extend(routes);
        self
    }

    pub async fn run(self) {
        // 1. Load config
        let config = AutumnConfig::load().unwrap_or_else(|e| {
            eprintln!("Failed to load configuration: {e}");
            std::process::exit(1);
        });

        // 2. Check routes
        assert!(!self.routes.is_empty(), "No routes registered. Did you forget to call .routes()?");

        // 3. Build Axum router
        let mut router = axum::Router::new();
        for route in &self.routes {
            println!("{} {} ({})", route.method, route.path, route.name);
            router = router.route(route.path, route.handler.clone());
        }
        let router = router.with_state(AppState);

        // 4. Bind and serve
        let addr = format!("{}:{}", config.server.host, config.server.port);
        println!("Listening on http://{addr}");

        let listener = tokio::net::TcpListener::bind(&addr)
            .await
            .expect("Failed to bind to address");

        axum::serve(listener, router)
            .with_graceful_shutdown(shutdown_signal())
            .await
            .expect("Server error");
    }
}

async fn shutdown_signal() {
    tokio::signal::ctrl_c()
        .await
        .expect("Failed to install Ctrl+C handler");
    println!("Shutting down...");
}
```

### Route.handler needs Clone

`MethodRouter<AppState>` must implement `Clone` for the router building loop. Axum's `MethodRouter` does implement `Clone`, so this should work.

### Testing Strategy

Unit testing a server is complex. For Sprint 3:
1. **Build test:** Verify `AppBuilder` compiles and routes can be added
2. **Panic test:** Verify empty routes panics (use `#[should_panic]`)
3. **Integration test (manual):** Run the example program, curl localhost:3000

Full integration tests with `reqwest` or `axum::test` can come in Sprint 4+.
