## User Story

As a **framework user**
I want to **see all mounted routes printed at startup and get a clear error if I forget to register any**
So that **I can verify my routes are correctly wired and catch configuration mistakes immediately**

---

## Description

### Background
S-009 implements the basic app builder with route mounting. This story refines the startup experience: structured route table output and a clear panic message when no routes are registered.

Note: S-009 already includes basic `println!` for routes and an `assert!` for empty routes. This story may be partially or fully satisfied by S-009's implementation. If so, this story becomes about adding tests and refining the output format.

### Scope
**In scope:**
- Formatted route table at startup (method, path, handler name)
- Clear panic message when zero routes are registered
- Tests verifying the panic behavior
- Output format should be scannable (aligned columns or consistent format)

**Out of scope:**
- Structured logging with `tracing` (S-028)
- Duplicate route detection (future)
- Route conflict detection (future)

---

## Acceptance Criteria

- [ ] At startup, each route is printed with method, path, and handler name
- [ ] Routes are printed in registration order
- [ ] If zero routes are registered, the app panics with "No routes registered. Did you forget to call .routes()?"
- [ ] The panic message is actionable (tells the user what to do)
- [ ] `#[should_panic]` test verifies the empty routes behavior
- [ ] Multiple `.routes()` calls are combined correctly

---

## Technical Notes

### Output Format

```
  GET /hello (hello)
 POST /users (create_user)
  GET /users/{id} (get_user)
  PUT /users/{id} (update_user)
```

Right-align methods for readability, or use a consistent simple format:
```
GET     /hello          hello
POST    /users          create_user
```

### Relationship to S-009

If S-009 already implements route logging and the empty-routes panic, this story becomes:
1. Verify the tests exist
2. Refine the output format if needed
3. Test edge cases (multiple `.routes()` calls, single route, many routes)

### Testing

```rust
#[test]
#[should_panic(expected = "No routes registered")]
fn empty_routes_panics() {
    // Build app with no routes and call run()
    // Need to use tokio runtime for the async run()
}
```
