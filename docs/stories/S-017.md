## User Story

As a **framework user**
I want to **declare `db: Db` in my handler signature and get an async database connection**
So that **I can query Postgres without manual pool management or boilerplate**

---

## Description

### Background

This is the extractor pattern — Autumn's core DX innovation inherited from Axum. A handler declares what it needs in its function signature, and the framework provides it. `Db` wraps a pooled `AsyncPgConnection`, acquired from the deadpool managed by `AppState`.

After this story, handlers look like:
```rust
#[get("/users")]
async fn list(db: Db) -> AutumnResult<Json<Vec<User>>> {
    let users = users::table.load(&mut *db).await?;
    Ok(Json(users))
}
```

This is the **3-month gut check** story — "Does the Db extractor work?"

### Scope

**In scope:**
- `Db` newtype wrapping the deadpool connection object
- `Deref`/`DerefMut` to `AsyncPgConnection` (so `&mut *db` gives a raw connection)
- `FromRequestParts<AppState>` impl that acquires a connection from the pool
- Rejection type: `AutumnError` with 503 Service Unavailable when pool is exhausted or no pool configured
- Re-export `Db` from `autumn` crate root
- Extend `autumn/src/db.rs` (created in S-016)
- Unit tests with mock/stub AppState (no live DB required for extractor logic)

**Out of scope:**
- Transaction helpers (`db.transaction(|conn| ...)`) — v1.0
- Read replicas or multiple pools — v1.0
- Connection health checking beyond deadpool's built-in — v1.0
- `tracing::error!` for pool failures (S-028 adds structured logging)

### The Extractor Flow

```
Request arrives
  → Axum calls FromRequestParts::from_request_parts()
    → Db::from_request_parts(parts, &AppState)
      → state.pool.get().await    // acquire connection from deadpool
        → Ok(conn) → Db(conn)    // wrap in newtype
        → Err(e)  → AutumnError with 503
  → Handler receives `db: Db`
  → Handler uses `&mut *db` as `&mut AsyncPgConnection`
  → Handler returns, Db dropped, connection returned to pool
```

## Technical Notes

### `Db` Type

```rust
// autumn/src/db.rs

use diesel_async::AsyncPgConnection;
use diesel_async::pooled_connection::deadpool::Pool;
use diesel_async::pooled_connection::AsyncDieselConnectionManager;

type PooledConnection = deadpool::managed::Object<
    AsyncDieselConnectionManager<AsyncPgConnection>
>;

/// Async database connection extractor.
///
/// Declare `db: Db` in your handler signature to get a pooled
/// connection to Postgres.
///
/// ```ignore
/// #[get("/users")]
/// async fn list(db: Db) -> AutumnResult<Json<Vec<User>>> {
///     let users = users::table.load(&mut *db).await?;
///     Ok(Json(users))
/// }
/// ```
///
/// The connection is returned to the pool when `Db` is dropped.
pub struct Db(PooledConnection);

impl std::ops::Deref for Db {
    type Target = AsyncPgConnection;
    fn deref(&self) -> &Self::Target {
        &self.0
    }
}

impl std::ops::DerefMut for Db {
    fn deref_mut(&mut self) -> &mut Self::Target {
        &mut self.0
    }
}
```

### `FromRequestParts` Implementation

```rust
use axum::extract::FromRequestParts;
use axum::http::StatusCode;

use crate::error::AutumnError;
use crate::AppState;

impl FromRequestParts<AppState> for Db {
    type Rejection = AutumnError;

    async fn from_request_parts(
        _parts: &mut http::request::Parts,
        state: &AppState,
    ) -> Result<Self, Self::Rejection> {
        let pool = state.pool.as_ref().ok_or_else(|| {
            AutumnError::bad_request(
                std::io::Error::new(std::io::ErrorKind::NotConnected, "Database not configured")
            ).with_status(StatusCode::SERVICE_UNAVAILABLE)
        })?;

        let conn = pool.get().await.map_err(|e| {
            eprintln!("Failed to acquire database connection: {e}");
            AutumnError::bad_request(
                std::io::Error::new(std::io::ErrorKind::Other, e.to_string())
            ).with_status(StatusCode::SERVICE_UNAVAILABLE)
        })?;

        Ok(Db(conn))
    }
}
```

**Note on `async_trait`:** Axum 0.8 no longer requires `#[async_trait]` for `FromRequestParts` — async trait methods are native. Verify this against the actual Axum 0.8 API.

### Testing Strategy

The extractor can be tested without a live database by testing the rejection paths:

```rust
#[tokio::test]
async fn db_extractor_rejects_when_no_pool() {
    // Build a router with no pool in AppState
    let app = Router::new()
        .route("/", get(|_db: Db| async { "ok" }))
        .with_state(AppState { pool: None });

    let response = app
        .oneshot(Request::builder().uri("/").body(Body::empty()).unwrap())
        .await
        .unwrap();

    assert_eq!(response.status(), StatusCode::SERVICE_UNAVAILABLE);
}
```

Testing the happy path (successful connection) requires a live Postgres. This can be deferred to CI integration tests. For now, verify:
1. The extractor compiles with correct types
2. Missing pool returns 503
3. `Deref`/`DerefMut` compile correctly
