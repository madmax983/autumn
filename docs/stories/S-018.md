## User Story

As a **framework user**
I want to **annotate a struct with `#[derive(Model)]` and have Diesel + Serde derives applied automatically**
So that **I can define database models with one attribute instead of five derive macros**

---

## Description

### Background

Diesel requires `#[derive(Queryable, Selectable)]` for reading and `#[derive(Insertable)]` for writing, plus `#[diesel(table_name = ...)]` attributes. Combined with Serde's `Serialize`/`Deserialize`, a typical model struct needs 5-6 derive macros and a table name attribute. `#[derive(Model)]` collapses this into a single annotation.

This is a **convenience macro** — it emits the struct with additional derive macros and Diesel attributes applied. The user's struct definition is preserved unchanged.

### Scope

**In scope:**
- `#[derive(Model)]` proc macro in `autumn-macros`
- `#[model(table = "users")]` attribute for explicit table name
- Inferred table name from struct name: `User` → `users`, `BlogPost` → `blog_posts` (snake_case + pluralize)
- Emit: `#[derive(Debug, Clone, Queryable, Selectable, Insertable, Serialize, Deserialize)]`
- Emit: `#[diesel(table_name = users)]`
- Simple pluralization (append 's') with `#[model(table = "...")]` override for irregular cases
- New file: `autumn-macros/src/model.rs`
- Compile-pass and compile-fail trybuild tests

**Out of scope:**
- Complex English pluralization ("person" → "people") — require explicit `table =` for irregular cases
- `NewUser` insertable variant generation (v0.2)
- Relation macros (`belongs_to`, `has_many`) — not in v0.1
- Diesel schema generation (`diesel print-schema`) — external tool
- Auto-generating Diesel `table!` macro calls — user provides schema.rs

### The Transformation

**Input:**
```rust
#[derive(Model)]
#[model(table = "users")]
pub struct User {
    pub id: i32,
    pub name: String,
    pub email: String,
}
```

**Output:**
```rust
#[derive(Debug, Clone, Queryable, Selectable, Insertable, Serialize, Deserialize)]
#[diesel(table_name = users)]
pub struct User {
    pub id: i32,
    pub name: String,
    pub email: String,
}
```

**Without explicit table:**
```rust
#[derive(Model)]
pub struct BlogPost { ... }
// Infers: #[diesel(table_name = blog_posts)]
```

## Technical Notes

### `autumn-macros/src/model.rs`

```rust
use proc_macro2::TokenStream;
use quote::quote;
use syn::{DeriveInput, LitStr};

pub fn model_macro(input: TokenStream) -> TokenStream {
    let input: DeriveInput = match syn::parse2(input) {
        Ok(input) => input,
        Err(err) => return err.to_compile_error(),
    };

    // Must be a struct
    if !matches!(input.data, syn::Data::Struct(_)) {
        return syn::Error::new_spanned(
            &input.ident,
            "#[derive(Model)] can only be applied to structs",
        )
        .to_compile_error();
    }

    // Extract #[model(table = "...")] if present
    let table_name = extract_table_name(&input.attrs)
        .unwrap_or_else(|| infer_table_name(&input.ident));

    let table_ident = syn::Ident::new(&table_name, input.ident.span());

    let name = &input.ident;
    let vis = &input.vis;
    let generics = &input.generics;
    let fields = match &input.data {
        syn::Data::Struct(data) => &data.fields,
        _ => unreachable!(),
    };

    // Filter out #[model(...)] from the output attributes
    let other_attrs: Vec<_> = input.attrs.iter()
        .filter(|attr| !attr.path().is_ident("model"))
        .collect();

    quote! {
        #[derive(Debug, Clone, ::diesel::Queryable, ::diesel::Selectable, ::diesel::Insertable)]
        #[derive(::serde::Serialize, ::serde::Deserialize)]
        #[diesel(table_name = #table_ident)]
        #(#other_attrs)*
        #vis struct #name #generics #fields
    }
}

fn extract_table_name(attrs: &[syn::Attribute]) -> Option<String> {
    for attr in attrs {
        if attr.path().is_ident("model") {
            // Parse: #[model(table = "users")]
            let mut table = None;
            attr.parse_nested_meta(|meta| {
                if meta.path.is_ident("table") {
                    let value: LitStr = meta.value()?.parse()?;
                    table = Some(value.value());
                }
                Ok(())
            }).ok()?;
            return table;
        }
    }
    None
}

fn infer_table_name(ident: &syn::Ident) -> String {
    // Convert PascalCase to snake_case and append 's'
    let name = ident.to_string();
    let snake = pascal_to_snake(&name);
    format!("{snake}s")
}

fn pascal_to_snake(s: &str) -> String {
    let mut result = String::new();
    for (i, ch) in s.chars().enumerate() {
        if ch.is_uppercase() && i > 0 {
            result.push('_');
        }
        result.push(ch.to_ascii_lowercase());
    }
    result
}
```

### Macro Registration

In `autumn-macros/src/lib.rs`:
```rust
mod model;

/// Derive macro for Autumn database models.
///
/// Applies Diesel (`Queryable`, `Selectable`, `Insertable`) and Serde
/// (`Serialize`, `Deserialize`) derives, plus a `#[diesel(table_name)]`
/// attribute.
///
/// # Example
///
/// ```ignore
/// use autumn_web::Model;
///
/// #[derive(Model)]
/// #[model(table = "users")]
/// pub struct User {
///     pub id: i32,
///     pub name: String,
/// }
/// ```
#[proc_macro_derive(Model, attributes(model))]
pub fn derive_model(input: TokenStream) -> TokenStream {
    model::model_macro(input.into()).into()
}
```

### Re-export from autumn

In `autumn/src/lib.rs`:

### Path Resolution for Diesel and Serde

The generated code uses `::diesel::Queryable`, `::serde::Serialize`, etc. This requires:
1. The user's crate has `diesel` and `serde` as dependencies (which they will if they're using `#[derive(Model)]`)
2. OR `autumn` re-exports them so the macro can use `::autumn_web::reexports::diesel`

**For v0.1:** Use absolute paths (`::diesel::`, `::serde::`) since the user needs these crates anyway for schema definitions and handler return types. Document the required dependencies.

**Future consideration:** Re-export diesel and serde from `autumn_web::reexports` so users only need `autumn` as a dependency.

### Table Name Inference

Simple pluralization rules:
- `User` → `users`
- `BlogPost` → `blog_posts`
- `Category` → `categorys` (wrong! but acceptable for v0.1 with `#[model(table = "categories")]` override)

Document that irregular plurals need explicit `#[model(table = "...")]`.

### Testing Challenges

`#[derive(Model)]` generates code that references `diesel::Queryable`, `diesel::Selectable`, etc. Compile-pass tests need a Diesel `table!` macro call to provide the schema. This means trybuild test files need diesel as a dependency.

**Approach:** Test via trybuild in the `autumn` crate (which has diesel as a dependency). The test file needs a `table!` macro:

```rust
// autumn/tests/compile-pass/model_derive.rs
use autumn_web::Model;

diesel::table! {
    users (id) {
        id -> Integer,
        name -> Text,
    }
}

#[derive(Model)]
#[model(table = "users")]
pub struct User {
    pub id: i32,
    pub name: String,
}

fn main() {}
```

The compile-fail tests are simpler — they don't need valid Diesel schemas:

```rust
// autumn/tests/compile-fail/model_on_enum.rs
use autumn_web::Model;

#[derive(Model)]
enum Status {
    Active,
    Inactive,
}

fn main() {}
```

## Additional Notes

- Architecture doc open issue #3 calls out pluralization: *"Pluralization in English is irregular. May need to use the `heck` crate or require explicit table names."* For v0.1, simple 's' suffix with explicit override is acceptable.
- The `heck` crate provides `ToSnakeCase` which is more robust than our custom `pascal_to_snake`. Consider adding it as a dependency to `autumn-macros` if the manual conversion has edge cases.
- This macro does NOT replace Diesel's schema. Users still need `diesel print-schema` to generate `schema.rs`. The `#[derive(Model)]` macro just reduces boilerplate on the struct definition.
- `#[derive(Model)]` is a **derive macro** (not an attribute macro like `#[get]`). This means it uses `#[proc_macro_derive(Model, attributes(model))]` syntax.
