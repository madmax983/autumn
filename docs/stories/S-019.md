## User Story

As a **framework user**
I want to **set my database URL in `autumn.toml` or via `AUTUMN_DATABASE__URL` and have the pool use it automatically**
So that **I can configure database access per-environment without touching code**

---

## Description

### Background

`DatabaseConfig` already exists with `url`, `pool_size`, and `connect_timeout_secs` fields, loaded from `autumn.toml` (S-026). S-016 creates the pool from this config. This story completes the integration by adding environment variable overrides (`AUTUMN_DATABASE__URL`, `AUTUMN_DATABASE__POOL_SIZE`) and validating the full config → pool → runtime pipeline.

The architecture doc specifies a three-layer config system:
1. Framework defaults (compiled in)
2. `autumn.toml` (project-level)
3. `AUTUMN_*` environment variables (deployment-level)

Layer 1 and 2 are done. This story adds layer 3 for database settings specifically, plus validation that the URL is well-formed before attempting pool creation.

### Scope

**In scope:**
- Environment variable overrides for database config:
  - `AUTUMN_DATABASE__URL` → `config.database.url`
  - `AUTUMN_DATABASE__POOL_SIZE` → `config.database.pool_size`
  - `AUTUMN_DATABASE__CONNECT_TIMEOUT_SECS` → `config.database.connect_timeout_secs`
- `AutumnConfig::apply_env_overrides()` method (or integrate into `load()`)
- Basic URL validation: must start with `postgres://` or `postgresql://`
- Integration test: set env var → load config → verify override
- Documentation of all `AUTUMN_*` database environment variables

**Out of scope:**
- Full `AUTUMN_*` env override system for all config sections (S-027 — separate story)
- Connection string parsing or parameter extraction
- SSL/TLS configuration
- Multiple database URLs / read replicas

### Environment Variable Convention

The double-underscore `__` separates nested config sections:
```
AUTUMN_DATABASE__URL       → config.database.url
AUTUMN_DATABASE__POOL_SIZE → config.database.pool_size
AUTUMN_SERVER__PORT        → config.server.port (future, S-027)
```

This convention avoids conflicts with single-underscore field names.

## Technical Notes

### Implementation Approach

Add an `apply_env_overrides()` method to `AutumnConfig`:

```rust
impl AutumnConfig {
    /// Apply environment variable overrides to the loaded config.
    ///
    /// Environment variables use double-underscore `__` to separate
    /// nested sections: `AUTUMN_DATABASE__URL`, `AUTUMN_SERVER__PORT`.
    pub fn apply_env_overrides(&mut self) {
        // Database overrides
        if let Ok(val) = std::env::var("AUTUMN_DATABASE__URL") {
            self.database.url = Some(val);
        }
        if let Ok(val) = std::env::var("AUTUMN_DATABASE__POOL_SIZE") {
            if let Ok(size) = val.parse::<usize>() {
                self.database.pool_size = size;
            } else {
                eprintln!(
                    "Warning: AUTUMN_DATABASE__POOL_SIZE={val:?} is not a valid number, ignoring"
                );
            }
        }
        if let Ok(val) = std::env::var("AUTUMN_DATABASE__CONNECT_TIMEOUT_SECS") {
            if let Ok(secs) = val.parse::<u64>() {
                self.database.connect_timeout_secs = secs;
            } else {
                eprintln!(
                    "Warning: AUTUMN_DATABASE__CONNECT_TIMEOUT_SECS={val:?} is not a valid number, ignoring"
                );
            }
        }
    }
}
```

### Wire into Config Loading

Update `AutumnConfig::load()` to apply env overrides:

```rust
pub fn load() -> Result<Self, ConfigError> {
    let mut config = Self::load_from(Path::new("autumn.toml"))?;
    config.apply_env_overrides();
    Ok(config)
}
```

### URL Validation

Add a `validate()` method to `DatabaseConfig` or `AutumnConfig`:

```rust
impl DatabaseConfig {
    /// Validate the database configuration.
    ///
    /// # Errors
    ///
    /// Returns an error if the database URL has an invalid scheme.
    pub fn validate(&self) -> Result<(), ConfigError> {
        if let Some(ref url) = self.url {
            if !url.starts_with("postgres://") && !url.starts_with("postgresql://") {
                return Err(ConfigError::Validation(format!(
                    "Invalid database URL: must start with postgres:// or postgresql://, got {url:?}"
                )));
            }
        }
        Ok(())
    }
}
```

Add a `Validation` variant to `ConfigError`:
```rust
#[derive(Debug, Error)]
pub enum ConfigError {
    #[error("failed to read autumn.toml: {0}")]
    Io(#[from] std::io::Error),
    #[error("invalid autumn.toml: {0}")]
    Parse(#[from] toml::de::Error),
    #[error("configuration error: {0}")]
    Validation(String),
}
```

### Testing Strategy

Environment variable tests need to be careful about global state. Use `std::env::set_var` / `std::env::remove_var` with unique prefixes, or use a test helper that takes explicit values.

```rust
#[test]
fn env_override_database_url() {
    // Save and restore
    let original = std::env::var("AUTUMN_DATABASE__URL").ok();
    std::env::set_var("AUTUMN_DATABASE__URL", "postgres://override:5432/test");

    let mut config = AutumnConfig::default();
    config.apply_env_overrides();

    assert_eq!(config.database.url.as_deref(), Some("postgres://override:5432/test"));

    // Restore
    match original {
        Some(val) => std::env::set_var("AUTUMN_DATABASE__URL", val),
        None => std::env::remove_var("AUTUMN_DATABASE__URL"),
    }
}
```

**Better approach:** Make `apply_env_overrides` accept a closure or trait for env var lookup, so tests can pass a mock. But for v0.1, direct `std::env` is fine.

Note: Env var tests must run sequentially (`cargo test -- --test-threads=1` for those specific tests) since `set_var` is process-global. Use `#[serial]` from `serial_test` crate if needed, or isolate in a separate test binary.

## Additional Notes

- This story implements database-specific env overrides only. The full `AUTUMN_*` system for all config sections is S-027 (Sprint 8). That story will generalize this pattern.
- The double-underscore convention matches tools like `config-rs` and Kubernetes env var injection patterns.
- `eprintln!` for invalid env var warnings will be replaced with `tracing::warn!` when structured logging (S-028) lands.
- Consider: should `AUTUMN_DATABASE_URL` (single underscore) also be checked? Many frameworks use `DATABASE_URL` directly. For now, stick with the `AUTUMN_` prefix convention for consistency.
