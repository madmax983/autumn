## User Story

I want **htmx to be available automatically without CDN links or manual script tags**
So that **`hx-get`, `hx-post`, `hx-swap` attributes work in my Maud templates out of the box**

---

## Description

### Background

htmx is Autumn's client-side interactivity layer. Instead of requiring a CDN link or npm install, Autumn embeds `htmx.min.js` directly in the crate via `include_bytes!` and serves it at a dedicated route. This means htmx is available offline, version-pinned, and zero-config.

The architecture specifies htmx is served via a **dedicated route** (not the static file serving middleware), since it's embedded in the binary, not on disk.

After this story:
```rust
use autumn::{get, html, Markup};

#[get("/")]
async fn index() -> Markup {
    html! {
        head {
            // htmx is served automatically at this path
            script src="/static/js/htmx.min.js" {}
        }
        body {
            button hx-get="/api/greeting" hx-swap="innerHTML" {
                "Click me"
            }
            // htmx just works — no CDN, no npm, no config
        }
    }
}
```

### Scope

**In scope:**
- Vendor htmx 2.x minified JS into `autumn/vendor/htmx.min.js`
- `include_bytes!("../vendor/htmx.min.js")` as a constant in the autumn crate
- Dedicated route at `/static/js/htmx.min.js` serving the embedded bytes
- Correct `Content-Type: application/javascript` and cache headers
- Integration test: GET `/static/js/htmx.min.js` returns valid JavaScript
- Document the htmx version and how to verify it

**Out of scope:**
- Automatic `<script>` tag injection into Maud templates (user adds it manually)
- htmx extensions (user-land concern)
- htmx configuration (`hx-boost`, `hx-history`, etc.) — user-land
- Feature flag to disable htmx (S-044, Sprint 11)
- Layout/base template system that auto-includes htmx

## Technical Notes

### Vendor the File

Download htmx 2.x minified:
```
curl -o autumn/vendor/htmx.min.js https://unpkg.com/htmx.org@2/dist/htmx.min.js
```

Verify integrity (document the SHA-256 hash in a comment or `.sha256` file).

Create `autumn/vendor/` directory and add `htmx.min.js`. Add a `autumn/vendor/README.md` noting the version and source URL.

### Embedding

In `autumn/src/lib.rs` or a new `autumn/src/htmx.rs`:

```rust
/// htmx 2.x embedded JavaScript.
///
/// Served automatically at `/static/js/htmx.min.js`.
/// Version: 2.0.4 (update this when upgrading)
pub(crate) const HTMX_JS: &[u8] = include_bytes!("../vendor/htmx.min.js");

/// htmx version string for diagnostics.
pub const HTMX_VERSION: &str = "2.0.4";
```

### Serving Route

In `autumn/src/app.rs`, register the htmx route in the app builder:

```rust
use axum::http::header;
use axum::response::{IntoResponse, Response};

async fn htmx_handler() -> Response {
    (
        [
            (header::CONTENT_TYPE, "application/javascript"),
            (header::CACHE_CONTROL, "public, max-age=31536000, immutable"),
        ],
        crate::htmx::HTMX_JS,
    )
        .into_response()
}

// In the router building step:
let router = router
    .route("/static/js/htmx.min.js", axum::routing::get(htmx_handler));
```

### Cache Headers

Since the content is immutable (pinned version, embedded at compile time), use aggressive caching:
- `Cache-Control: public, max-age=31536000, immutable`
- This tells browsers and CDNs to cache for 1 year and never revalidate

When htmx is upgraded in a future Autumn release, the content changes with the binary — no cache busting needed since the binary itself is different.

### Route Path Choice

`/static/js/htmx.min.js` was chosen to:
1. Match the convention for static assets (`/static/...`)
2. Be predictable for users writing `<script src="...">` tags
3. Not conflict with user-defined routes (users own `/api/`, etc.)
4. Not conflict with S-032's `ServeDir` for `static/` on disk (the embedded route takes priority)

### htmx 2.x Notes

htmx 2.x key differences from 1.x:
- Drops IE11 support (fine for Autumn)
- `hx-on` attribute syntax changed
- Default swap delay changed
- Some extension APIs changed

Pin to the latest 2.0.x stable. Document the exact version.

## Additional Notes

- The `vendor/` directory should be gitignored for user projects but committed in the Autumn repo. Autumn's `vendor/` is part of the crate source.
- `include_bytes!` means the htmx JS is compiled into the binary. This adds ~45KB (minified htmx 2.x) to the binary size — acceptable.
- Future consideration: serve htmx with `ETag` header based on a compile-time hash. For v0.1, `Cache-Control: immutable` is sufficient.
- Users who don't want htmx can simply not include the `<script>` tag. The route exists but costs nothing if not requested. Feature flags (S-044) will allow removing it from the binary entirely.
