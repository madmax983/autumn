## User Story

As a **framework user**
I want **a starter `input.css` with Tailwind directives included in my project**
So that **`cargo build` produces a working stylesheet without me needing to know Tailwind's configuration syntax**

---

## Description

### Background

Tailwind CSS requires an input file that contains its directives (`@import "tailwindcss"`). The build.rs template (S-021) reads `static/css/input.css` and outputs `static/css/autumn.css`. This story creates the starter input.css that ships with new Autumn projects.

Tailwind v4 moved to CSS-based configuration — `@import "tailwindcss"` replaces the old `@tailwind base; @tailwind components; @tailwind utilities;` directives. The input file can also contain custom CSS and Tailwind theme customizations.

After this story, a new Autumn project has:
```
static/
└── css/
    ├── input.css    ← this file (checked in, user-editable)
    └── autumn.css   ← generated by build.rs (gitignored)
```

### Scope

**In scope:**
- `autumn/templates/static/css/input.css` template file
- Tailwind v4 `@import "tailwindcss"` directive
- Comments explaining what the file does and how to customize it
- Autumn-specific base styles (CSS reset is handled by Tailwind's preflight)

**Out of scope:**
- Custom Tailwind theme configuration (user-land)
- `tailwind.config.js` (Tailwind v4 uses CSS-based config)
- Building or running Tailwind (S-021)
- The output `autumn.css` file (generated, not checked in)

## Technical Notes

### Template Content

```css
/*
 * Autumn — Tailwind CSS input file
 *
 * This file is processed by the Tailwind CLI during `cargo build`.
 * The output lands at static/css/autumn.css (which is gitignored).
 *
 * Add your custom styles below the @import. Tailwind utility classes
 * used in your Maud templates (src/**/*.rs) are automatically included.
 *
 * Tailwind v4 docs: https://tailwindcss.com/docs
 */

@import "tailwindcss";

/*
 * Custom styles
 * ─────────────
 * Add project-specific styles here. They'll be included in autumn.css
 * alongside the Tailwind utilities your templates use.
 *
 * Example:
 *
 * .prose-autumn {
 *     @apply prose prose-lg prose-slate;
 * }
 */
```

### Tailwind v4 CSS-Based Config

Tailwind v4 replaced `tailwind.config.js` with CSS-based configuration using `@theme`:

```css
@import "tailwindcss";

@theme {
  --color-autumn: #c0570a;
  --font-display: "Cal Sans", sans-serif;
}
```

The starter file doesn't include `@theme` customization — that's user territory. The comments point users to the docs.

### File Location

The template lives at `autumn/templates/static/css/input.css`. This mirrors the path it will have in user projects (`static/css/input.css`). The `autumn new` command (S-035) copies it verbatim.

### .gitignore Entry

User projects should gitignore the output but not the input:
```
# Tailwind output (generated by build.rs)
static/css/autumn.css
```

This will be part of the `.gitignore` template in S-035. For now, just note it in the story.

## Additional Notes

- Tailwind v4's `@import "tailwindcss"` is equivalent to v3's three-directive pattern (`@tailwind base/components/utilities`). It's simpler and more intuitive.
- The input.css is the user's customization point for the CSS pipeline. Keep the starter minimal — don't add opinionated styles that users will immediately delete.
- Future: an `autumn` Tailwind plugin could provide framework-specific utilities. Out of scope for v0.1.
