## Description

### Background
Autumn uses a three-layer configuration system: framework defaults (compiled in) -> `autumn.toml` -> `AUTUMN_*` environment variables. This story defines the foundation — the `AutumnConfig` struct hierarchy with `serde::Deserialize` and `#[serde(default)]` annotations so that every field has a sensible default value.

This is the "productive procrastination" task for Sprint 1. It's pure data structures — no proc macros, no runtime logic, no external dependencies beyond serde. When proc macro work on S-002 gets frustrating, this is the context switch.

### Scope
**In scope:**
- `AutumnConfig` top-level struct
- `ServerConfig` (port, host, shutdown_timeout_secs)
- `DatabaseConfig` (url, pool_size, connect_timeout_secs)
- `LogConfig` (level, format)
- `HealthConfig` (path)
- `LogFormat` enum (Auto, Pretty, Json)
- All `#[serde(default = "...")]` annotations with default functions
- `Default` impl for all config types
- Unit tests verifying all defaults

**Out of scope:**
- TOML file loading (`autumn.toml` parsing) — S-026
- Environment variable overrides (`AUTUMN_*`) — S-027
- Wiring config into AppState or App builder — S-009, S-019
- Logging setup using these config values — S-028
- Any runtime behavior — this is just types + defaults

## Technical Notes

### Config Module Structure

```rust
// autumn/src/config.rs

use serde::Deserialize;

#[derive(Debug, Deserialize)]
pub struct AutumnConfig {
    #[serde(default)]
    pub server: ServerConfig,
    #[serde(default)]
    pub database: DatabaseConfig,
    #[serde(default)]
    pub log: LogConfig,
    #[serde(default)]
    pub health: HealthConfig,
}

#[derive(Debug, Deserialize)]
pub struct ServerConfig {
    #[serde(default = "default_port")]
    pub port: u16,
    #[serde(default = "default_host")]
    pub host: String,
    #[serde(default = "default_shutdown_timeout")]
    pub shutdown_timeout_secs: u64,
}

#[derive(Debug, Deserialize)]
pub struct DatabaseConfig {
    #[serde(default = "default_db_url")]
    pub url: String,
    #[serde(default = "default_pool_size")]
    pub pool_size: usize,
    #[serde(default = "default_connect_timeout")]
    pub connect_timeout_secs: u64,
}

#[derive(Debug, Deserialize)]
pub struct LogConfig {
    #[serde(default = "default_log_level")]
    pub level: String,
    #[serde(default)]
    pub format: LogFormat,
}

#[derive(Debug, Deserialize, Default)]
pub enum LogFormat {
    #[default]
    Auto,
    Pretty,
    Json,
}

#[derive(Debug, Deserialize)]
pub struct HealthConfig {
    #[serde(default = "default_health_path")]
    pub path: String,
}
```

### Default Functions

```rust
fn default_port() -> u16 { 3000 }
fn default_host() -> String { "127.0.0.1".to_owned() }
fn default_shutdown_timeout() -> u64 { 30 }
fn default_db_url() -> String { "postgres://localhost/autumn_dev".to_owned() }
fn default_pool_size() -> usize { 10 }
fn default_connect_timeout() -> u64 { 5 }
fn default_log_level() -> String { "info".to_owned() }
fn default_health_path() -> String { "/health".to_owned() }
```

### Design Decisions

- **Named default functions, not `Default` trait only:** Serde's `#[serde(default)]` at the field level uses named functions, which is more explicit than relying on the `Default` trait. Both are implemented — `Default` for programmatic use, serde defaults for deserialization.
- **`String` for log level, not an enum:** Allows users to set arbitrary tracing filter directives (e.g., `"autumn=debug,tower_http=trace"`). Validated at runtime when `tracing-subscriber` is initialized, not at config parse time.
- **`LogFormat` is an enum:** Only three valid options — `Auto` (environment-dependent), `Pretty`, `Json`. Invalid values should produce a serde error.
- **`u64` for timeouts, not `Duration`:** Serde doesn't natively deserialize `Duration` from TOML integers. `u64` seconds is the simplest representation. Convert to `Duration` at usage sites.
