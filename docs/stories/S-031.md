## User Story

As a **framework user**
I want to **use `Json<T>` in my handler signatures and return types without depending on axum directly**
So that **I can build JSON APIs with `use autumn::extract::Json` and a clean handler signature**

---

## Description

### Background

Axum's `Json<T>` serves double duty — it's both an extractor (parses JSON request bodies) and a response type (serializes to JSON with `application/json` content type). Autumn re-exports it so users don't need `axum` as a direct dependency.

The extract module already has `Path`, `Query`, and (after S-023) `Form`. This story adds `Json` and verifies both extractor and response behavior.

After this story:
```rust
use autumn::{get, post, extract::Json, AutumnResult};
use serde::{Deserialize, Serialize};

#[derive(Serialize)]
struct User { name: String }

#[derive(Deserialize)]
struct CreateUser { name: String }

#[get("/users")]
async fn list() -> Json<Vec<User>> {
    Json(vec![User { name: "Alice".into() }])
}

#[post("/users")]
async fn create(Json(data): Json<CreateUser>) -> AutumnResult<Json<User>> {
    Ok(Json(User { name: data.name }))
}
```

### Scope

**In scope:**
- Re-export `axum::Json` from `autumn::extract` (and optionally from `autumn` root)
- Verify `Json<T>` works as an extractor (request body parsing)
- Verify `Json<T>` works as a response type (serialization + content type)
- Verify `AutumnResult<Json<T>>` works as a return type
- Integration test: POST JSON body, return JSON response
- Integration test: verify `application/json` content type on response

**Out of scope:**
- Custom JSON error format for parse failures (use Axum's default)
- JSON streaming
- JSON Patch / Merge Patch

## Technical Notes

### Implementation

One line in `autumn/src/extract.rs`:

```rust
pub use axum::extract::{Form, Json, Path, Query};
```

Also consider a root-level re-export since `Json` is used in return types too:
```rust
// autumn/src/lib.rs
pub use axum::Json;
```

This lets users write `autumn::Json<T>` in return position without importing from `extract`.

### Testing

```rust
#[tokio::test]
async fn json_extractor_and_response() {
    use axum::Router;
    use axum::body::Body;
    use axum::http::{Request, StatusCode};
    use autumn::extract::Json;
    use serde::{Deserialize, Serialize};
    use tower::ServiceExt;

    #[derive(Deserialize)]
    struct Input { value: i32 }

    #[derive(Serialize)]
    struct Output { doubled: i32 }

    async fn handler(Json(input): Json<Input>) -> Json<Output> {
        Json(Output { doubled: input.value * 2 })
    }

    let app = Router::new()
        .route("/", axum::routing::post(handler));

    let response = app
        .oneshot(
            Request::builder()
                .method("POST")
                .uri("/")
                .header("content-type", "application/json")
                .body(Body::from(r#"{"value": 21}"#))
                .unwrap(),
        )
        .await
        .unwrap();

    assert_eq!(response.status(), StatusCode::OK);

    let content_type = response.headers()
        .get("content-type")
        .unwrap()
        .to_str()
        .unwrap();
    assert!(content_type.contains("application/json"));

    let body = axum::body::to_bytes(response.into_body(), usize::MAX)
        .await
        .unwrap();
    let output: serde_json::Value = serde_json::from_slice(&body).unwrap();
    assert_eq!(output["doubled"], 42);
}

#[tokio::test]
async fn invalid_json_returns_422() {
    async fn handler(Json(_): Json<serde_json::Value>) -> &'static str { "ok" }

    let app = Router::new()
        .route("/", axum::routing::post(handler));

    let response = app
        .oneshot(
            Request::builder()
                .method("POST")
                .uri("/")
                .header("content-type", "application/json")
                .body(Body::from("not json"))
                .unwrap(),
        )
        .await
        .unwrap();

    // Axum returns 422 for invalid JSON
    assert_eq!(response.status(), StatusCode::UNPROCESSABLE_ENTITY);
}
```
