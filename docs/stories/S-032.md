## User Story

As a **framework user**
I want **files in my `static/` directory to be served automatically at `/static/`**
So that **CSS, images, and other assets are accessible without writing route handlers**

---

## Description

### Background

Autumn's app builder automatically serves files from the project's `static/` directory at the `/static/` URL path. This is how the Tailwind-generated CSS (`static/css/autumn.css`) reaches the browser, and how users serve images, fonts, and other static assets.

The architecture uses `tower-http`'s `ServeDir` middleware, which handles MIME type detection, range requests, and conditional responses (ETag/If-Modified-Since).

The embedded htmx route (S-022) at `/static/js/htmx.min.js` takes priority over the directory — if a user places a file at that path, the embedded version wins. This is intentional: the framework-provided htmx is always the pinned version.

After this story:
```
static/
├── css/
│   ├── input.css      ← user-editable Tailwind input
│   └── autumn.css     ← generated by build.rs
├── images/
│   └── logo.png       ← user file
└── js/
    └── app.js         ← user file
```

All accessible at `/static/css/autumn.css`, `/static/images/logo.png`, etc.

### Scope

**In scope:**
- Add `tower-http` dependency with `fs` feature
- `ServeDir` middleware mounted at `/static` pointing to `static/` directory
- Automatic registration in the app builder
- Graceful handling when `static/` directory doesn't exist (no panic, just 404s)
- Integration test: place a file, request it, verify content and MIME type
- Correct ordering: embedded htmx route takes priority over ServeDir

**Out of scope:**
- Cache headers for static files (use tower-http defaults for v0.1)
- Content-Security-Policy headers (S-045 area, v1.0)
- Asset fingerprinting / cache busting (v1.0)
- Compression (gzip/brotli) middleware (v1.0)
- Custom static directory path configuration (always `static/` for v0.1)

## Technical Notes

### Dependencies

Add to workspace `Cargo.toml`:
```toml
tower-http = { version = "0.6", features = ["fs"] }
```

Add to `autumn/Cargo.toml`:
```toml
tower-http.workspace = true
```

### Implementation

In `autumn/src/app.rs`, add ServeDir to the router:

```rust
use tower_http::services::ServeDir;

// In the router building step (after user routes, before framework middleware):
let router = router
    // Embedded framework routes (htmx) registered first — they take priority
    .route("/static/js/htmx.min.js", axum::routing::get(htmx_handler))
    // Then static directory fallback
    .nest_service("/static", ServeDir::new("static"));
```

### Route Priority

Axum's router gives explicit routes priority over nested services. This means:
1. `/static/js/htmx.min.js` → hits the embedded htmx handler (explicit route)
2. `/static/css/autumn.css` → hits ServeDir (nested service fallback)
3. `/static/anything/else` → hits ServeDir

This is the desired behavior. The embedded htmx is always the framework-pinned version.

### Missing Directory Handling

`ServeDir` returns 404 for paths that don't resolve to a file. If the `static/` directory itself doesn't exist, all requests to `/static/` return 404. This is fine — not all projects need static files (API-only projects).

No special handling needed. Don't panic, don't warn. Just 404.

### Integration Test

```rust
#[tokio::test]
async fn static_files_are_served() {
    use std::fs;
    use axum::body::Body;
    use axum::http::{Request, StatusCode};
    use tower::ServiceExt;

    // Create a temp static directory with a test file
    let dir = tempfile::tempdir().unwrap();
    let css_dir = dir.path().join("css");
    fs::create_dir_all(&css_dir).unwrap();
    fs::write(css_dir.join("test.css"), "body { color: red; }").unwrap();

    // Build router with ServeDir pointing to temp dir
    let app = Router::new()
        .nest_service("/static", ServeDir::new(dir.path()));

    let response = app
        .oneshot(
            Request::builder()
                .uri("/static/css/test.css")
                .body(Body::empty())
                .unwrap(),
        )
        .await
        .unwrap();

    assert_eq!(response.status(), StatusCode::OK);

    let content_type = response.headers()
        .get("content-type")
        .unwrap()
        .to_str()
        .unwrap();
    assert!(content_type.contains("text/css"));

    let body = axum::body::to_bytes(response.into_body(), usize::MAX)
        .await
        .unwrap();
    assert_eq!(&body[..], b"body { color: red; }");
}
```
