## User Story

As a **Rust developer discovering Autumn**
I want **a CLI tool I can install with `cargo install autumn-cli`**
So that **I can scaffold and manage Autumn projects without manually setting up boilerplate**

---

## Description

### Background

Autumn is feature-complete on the runtime side (Sprint 8 milestone). The next barrier to adoption is the "first 60 seconds" experience — how quickly a developer goes from zero to a running app. This story creates the CLI binary crate that will host the `autumn new` scaffolding command (S-035) and `autumn setup` tool management (S-036).

The CLI is architecturally independent from the framework runtime (TD-006): `autumn-cli` has **zero dependency on the `autumn` crate**. It generates projects from templates via string substitution, not from framework types. This keeps `cargo install autumn-cli` fast — no Axum, Diesel, or Maud in the CLI's dependency tree.

### Scope

**In scope:**
- New `autumn-cli/` workspace member with `Cargo.toml`
- Binary named `autumn` (via `[[bin]]` section)
- clap derive-based CLI with `--version` and `--help`
- `Commands` enum with `New { name }` and `Setup` subcommands
- `new` subcommand: stub implementation (S-035 will fill it in)
- `setup` subcommand: stub with helpful message directing to manual install
- Unit tests for CLI argument parsing
- Workspace root `Cargo.toml` updated to include `autumn-cli`

**Out of scope:**
- Actual project generation logic (S-035)
- Tailwind CLI download (S-036)
- Diesel migration runner (future)
- Shell completions
- Config file for CLI itself

## Technical Notes

### Crate Structure

```
autumn-cli/
  Cargo.toml
  src/
    main.rs       # Cli struct, Commands enum, dispatch
    new.rs        # pub fn run(name: &str) — stub for S-035
    setup.rs      # pub fn run() — helpful stub message
```

### autumn-cli/Cargo.toml

```toml
[package]
name = "autumn-cli"
description = "CLI tool for the Autumn web framework"
edition.workspace = true
version.workspace = true
license.workspace = true
repository.workspace = true

[[bin]]
name = "autumn"
path = "src/main.rs"

[dependencies]
clap = { version = "4", features = ["derive"] }
thiserror = "2"

[lints]
workspace = true
```

Key: `clap` is local to this crate (no other crate needs it). `thiserror` for error types in S-035.

### CLI Design (src/main.rs)

```rust
use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "autumn", version, about = "The Autumn web framework CLI")]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    /// Create a new Autumn project
    New {
        /// Project name (must be a valid Rust package name)
        name: String,
    },
    /// Download and configure external tools (Tailwind CSS)
    Setup,
}
```

### Testing

Use `Cli::try_parse_from()` for argument parsing tests:

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn parse_new_subcommand() {
        let cli = Cli::try_parse_from(["autumn", "new", "my-app"]).unwrap();
        assert!(matches!(cli.command, Commands::New { ref name } if name == "my-app"));
    }

    #[test]
    fn parse_setup_subcommand() {
        let cli = Cli::try_parse_from(["autumn", "setup"]).unwrap();
        assert!(matches!(cli.command, Commands::Setup));
    }

    #[test]
    fn no_args_is_error() {
        assert!(Cli::try_parse_from(["autumn"]).is_err());
    }
}
```

## Additional Notes

- The binary is named `autumn` (not `autumn-cli`) so users run `autumn new my-app`. The crate/package name remains `autumn-cli` for clarity on crates.io.
- `clap` v4 with derive feature is the standard choice for Rust CLI tools. It provides `--help`, `--version`, and argument validation for free.
- The `setup` stub is intentionally user-friendly — it points to manual Tailwind installation rather than silently failing. This ensures the scaffolded project (S-035) can work even before `autumn setup` is implemented.
