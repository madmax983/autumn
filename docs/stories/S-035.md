## User Story

As a **developer trying Autumn for the first time**
I want **`autumn new my-app` to generate a complete, ready-to-run project**
So that **I can have a working web app in under 5 minutes without manual setup**

---

## Description

### Background

The `autumn new` command is the single most important adoption moment (PRD EPIC-001). This is the `spring init` / `rails new` equivalent. If it doesn't work flawlessly, developers never see the rest of the framework.

This story implements the project generation logic in the `autumn-cli` crate (created in S-034). The CLI generates a complete project directory from embedded templates using simple string substitution. The generated project includes sample routes, configuration, Tailwind CSS build pipeline, and directory structure — everything needed to `cargo run` and see a working app.

Per architecture decision TD-006, the CLI does NOT depend on the `autumn` runtime crate. Templates are embedded in the binary via `include_str!()` and use placeholder substitution (`{{project_name}}`, `{{crate_name}}`, `{{autumn_version}}`).

### Scope

**In scope:**
- Project name validation (valid Rust package name)
- Directory creation with full project structure
- Template files embedded via `include_str!()`
- String substitution for project name, crate name, version
- Generated files: Cargo.toml, src/main.rs, autumn.toml, build.rs, .gitignore, input.css, tailwind.config.js, migrations/.gitkeep
- Clear error messages for invalid names and existing directories
- Success output with "next steps" instructions
- Unit tests for name validation, file generation, and template content

**Out of scope:**
- `cargo build` verification of generated project (S-037 E2E test)
- Tailwind CLI download during scaffolding (S-036)
- Database migration generation (future)
- Interactive prompts (e.g., "choose database type")
- Git init in generated project
- Template engine dependency (no Tera/Handlebars — plain `str::replace()`)

### Sprint Notes

S-035 depends on S-034 (the CLI crate must exist). The generated project references `autumn = "0.1"` as a crates.io dependency. During Sprint 9 development, we cannot verify `cargo build` of the generated project since `autumn` isn't published yet — that verification is explicitly S-037 (Sprint 10).

## Technical Notes

### Template Strategy

Templates live in `autumn-cli/src/templates/` as `.tmpl` files. They are compiled into the binary:

```rust
mod templates {
    pub const CARGO_TOML: &str = include_str!("templates/Cargo.toml.tmpl");
    pub const MAIN_RS: &str = include_str!("templates/main.rs.tmpl");
    pub const AUTUMN_TOML: &str = include_str!("templates/autumn.toml.tmpl");
    pub const BUILD_RS: &str = include_str!("templates/build.rs.tmpl");
    pub const INPUT_CSS: &str = include_str!("templates/input.css.tmpl");
    pub const TAILWIND_CONFIG: &str = include_str!("templates/tailwind.config.js.tmpl");
    pub const GITIGNORE: &str = include_str!("templates/gitignore.tmpl");
}
```

### Placeholder Substitution

Three placeholders, applied via `str::replace()`:
- `{{project_name}}` — the name as provided (e.g., `my-app`)
- `{{crate_name}}` — underscored for Rust identifiers (e.g., `my_app`)
- `{{autumn_version}}` — from `env!("CARGO_PKG_VERSION")`

### Project Name Validation

Must be a valid Rust package name:
- Non-empty
- Starts with ASCII letter
- Contains only ASCII alphanumeric, hyphens, or underscores
- Not a Rust keyword

Implemented as a simple char-by-char check — no regex dependency.

### Generated Project Structure

```
my-app/
  Cargo.toml              # depends on autumn = "0.1"
  autumn.toml             # default config (port 3000, info logging)
  build.rs                # Tailwind CLI runner with PATH fallback
  .gitignore              # /target, static/css/autumn.css
  src/
    main.rs               # 3 sample routes + #[autumn::main]
  static/
    css/
      input.css           # @tailwind base/components/utilities
  tailwind.config.js      # content: ["src/**/*.rs"]
  migrations/
    .gitkeep
```

### Output Directory Parameter

The `run()` function takes `parent_dir: &Path` to avoid CWD dependency in tests:

```rust
pub fn run(name: &str, parent_dir: &Path) -> Result<(), NewError> { ... }
```

In `main.rs`: `new::run(&name, &std::env::current_dir()?)`.

### Error Type

```rust
#[derive(Debug, thiserror::Error)]
pub enum NewError {
    #[error("invalid project name '{0}': {1}")]
    InvalidName(String, String),
    #[error("directory '{0}' already exists")]
    AlreadyExists(String),
    #[error("failed to create project: {0}")]
    Io(#[from] std::io::Error),
}
```

### Template Contents Reference

**Generated main.rs** mirrors `examples/hello/src/main.rs`:
```rust
use autumn::{get, routes};

#[get("/")]
async fn index() -> &'static str {
    "Welcome to Autumn!"
}

#[get("/hello")]
async fn hello() -> &'static str {
    "Hello, Autumn!"
}

#[get("/hello/{name}")]
async fn hello_name(name: autumn::extract::Path<String>) -> String {
    format!("Hello, {}!", *name)
}

#[autumn::main]
async fn main() {
    autumn::app()
        .routes(routes![index, hello, hello_name])
        .run()
        .await;
}
```

**Generated autumn.toml** must match defaults from `autumn/src/config.rs`.

## Additional Notes

- The generated `Cargo.toml` references `autumn = "0.1"` which won't resolve until `autumn` is published to crates.io (Sprint 12). During development, the E2E test (S-037) will use a path override. Sprint 9 tests verify file content only, not compilation.
- The `build.rs` template includes an inline `which()` function rather than depending on the `which` crate. This keeps the generated project's dependency tree minimal.
- Generated `main.rs` uses direct imports (`use autumn::{get, routes}`) rather than the prelude (`use autumn::prelude::*`) — the prelude is more useful for larger projects, but explicit imports are clearer for a starter template.
- Future enhancement: `autumn new --no-tailwind` flag to skip CSS pipeline. Not needed for v0.1.
