## User Story

As a **developer discovering Autumn on GitHub or crates.io**
I want **a README that shows me what Autumn is, how to start, and what to expect**
So that **I can decide in 60 seconds whether to try it and get a running app in 5 minutes**

---

## Description

### Background

The README is the single most important adoption artifact. Research showed that community credibility depends on documentation quality — competing frameworks were criticized for sparse docs. The README is often the only chance to convert a visitor into a user.

This story creates a README that is honest about maturity (v0.1 experimental), clear about what Autumn offers (Spring Boot-style DX for Rust), and provides a copy-pasteable quickstart that actually works.

### Scope

**In scope:**
- One-paragraph elevator pitch
- Feature list (what Autumn provides)
- Quickstart: install CLI → scaffold → run → see result (5 steps max)
- Example code showing a handler with route macro
- Honest maturity warning ("v0.1 — experimental, API will change")
- Technology acknowledgments (Axum, Diesel, Maud, htmx, Tailwind)
- Links to getting started guide and API docs (placeholders until Sprint 11)
- License section
- MSRV badge

**Out of scope:**
- Full getting started guide (S-039)
- Tutorial content (S-040)
- API reference (S-042)
- Contributing guidelines (post-v0.1)
- Benchmarks or performance claims
- Comparison with other frameworks

### Tone Guidelines

Per PRD FR-031:
- **DO:** Be clear, honest, technically accurate
- **DO:** Mention Autumn's relationship to Axum, Diesel, Maud, htmx, Tailwind
- **DON'T:** Use "blazing fast," "production-ready," or "enterprise-grade"
- **DON'T:** Oversell what v0.1 delivers
- **DO:** Include prominent maturity warning

### Quickstart Content

The quickstart must match what `autumn new` actually generates. Test by copy-pasting into a terminal.

```bash
# Install the CLI
cargo install autumn-cli

# Create a new project
autumn new my-app
cd my-app

# (Optional) Download Tailwind CSS
autumn setup

# Run it
cargo run

# Visit http://localhost:3000
```

## Additional Notes

- The README should be written AFTER the example app (S-041) lands, so it can reference a real working example. However, Sprint 10 has both stories — write README last in the sprint.
- crates.io uses README.md as the crate landing page. This is the first thing a potential user sees.

- The `cargo install autumn-cli` step will only work after crates.io publication (S-045, Sprint 12). Until then, the quickstart works from a git clone. Note this in the README if needed.
