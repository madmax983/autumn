## User Story

As a **Rust developer discovering Autumn for the first time**
I want **a step-by-step guide that takes me from zero to a running web app**
So that **I can evaluate Autumn quickly without reading source code or guessing at conventions**

---

## Description

### Background

The README (S-038) gives a feature overview and a quickstart snippet, but it's not enough for someone to actually build something. The getting started guide is the bridge between "this looks interesting" and "I shipped a page." It walks through every step — installation, project creation, adding routes, connecting a database, rendering HTML — at a pace that lets the reader understand *why* each piece exists, not just copy-paste.

This guide targets a Rust developer who knows `cargo` but has never touched Autumn. They may or may not know Axum, Diesel, or Maud. The guide should be self-contained — no "see the Axum docs" without explaining the relevant concept first.

### Scope

**In scope:**
- `docs/guide/getting-started.md` (primary artifact)
- Installation: `cargo install autumn-cli` (or `cargo install --path autumn-cli` for local dev)
- `autumn new my-app` walkthrough — what each generated file does
- `autumn setup` — Tailwind CLI download
- Adding a route: `#[get("/")]` with explanation of the macro
- Running the dev server: `cargo run`, visiting localhost
- Adding a second route with a path parameter
- Database setup: docker-compose, Diesel migration, `Db` extractor
- Maud template: rendering HTML with Tailwind classes
- htmx: adding interactivity (one example — e.g., inline delete)
- Configuration: `autumn.toml` explanation
- Error handling: using `AutumnResult<T>` and `?`
- Links to deeper docs (API reference, tutorial, example app)

**Out of scope:**
- Production deployment
- Advanced configuration (env overrides, custom logging)
- Testing strategies
- The full tutorial (that's S-040) — this guide is shorter and faster-paced
- API reference (that's S-042)

### User Flow

1. Reader installs the CLI (`cargo install autumn-cli`)
2. Reader runs `autumn new my-app && cd my-app`
3. Guide explains the generated project structure
4. Reader adds a hello-world route, runs `cargo run`, sees output in browser
5. Reader adds a route with a path parameter (`/hello/{name}`)
6. Reader sets up Postgres via docker-compose
7. Reader runs a Diesel migration and adds a model
8. Reader writes a route that queries the database and returns Maud HTML
9. Reader adds an htmx attribute for one interactive action
10. Reader has a working, styled, database-backed web app

### Tone

Conversational but efficient. No "In this section we will learn about..." — just do it. Think Axum's example-first style crossed with Rails Getting Started's completeness. Code first, explanation after.

### Code Examples Must Match Current API

All code examples should use the actual current public API. Key patterns:

```rust
use autumn_web::prelude::*;

#[get("/")]
async fn index() -> &'static str {
    "Hello, Autumn!"
}

#[get("/hello/{name}")]
async fn hello(Path(name): Path<String>) -> String {
    format!("Hello, {name}!")
}

#[get("/todos")]
async fn list_todos(db: Db) -> AutumnResult<Markup> {
    // ... query and render
}

#[autumn_web::main]
async fn main() {
    autumn_web::app()
        .routes(routes![index, hello, list_todos])
        .run()
        .await;
}
```

## Additional Notes

- The guide should be tested by following it from scratch on a clean machine (or at least a clean directory). If any step doesn't work, the guide is wrong.
- Avoid explaining Rust basics (ownership, borrowing, async) — the reader knows Rust. Explain Autumn concepts and conventions.
- Use the actual `autumn_web::prelude::*` import — don't import individual items unless explaining what the prelude contains.
- Screenshots are nice-to-have but not required. Terminal output is sufficient.
- The guide length should be ~1500-2500 words. Long enough to be complete, short enough to finish in one sitting.
