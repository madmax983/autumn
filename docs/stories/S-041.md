## Description

### Background

The example app serves three purposes: (1) documentation — developers learn by reading real code, (2) integration test — if the example compiles and runs, the framework works, (3) credibility — a working app proves Autumn delivers on its promise.

The architecture specifies a `examples/todo-app/` directory with multiple route modules, database queries, Maud templates with Tailwind and htmx, and a JSON API. This is the "reference application" that the tutorial (S-040) will build towards.

The existing `examples/hello/` is a minimal smoke test. This example is the full showcase.

### Scope

**In scope:**
- `examples/todo-app/` workspace member
- Two resource types: Todos and (optionally) Categories/Tags
- Routes: list, get by ID, create (via form and JSON API)
- Database: Diesel models, migrations, queries against Postgres (the framework's target DB)
- Maud templates with Tailwind CSS classes
- htmx attributes for interactive behavior (e.g., inline create, delete without page reload)
- Form submissions that write to the database
- JSON API endpoints (`/api/todos`)
- Error handling (404 for missing resource, validation for empty title)
- Configuration via `autumn.toml`
- Health check endpoint active
- Compiles and runs as standalone app

**Out of scope:**
- Authentication / user accounts
- Full CRUD (update can be deferred — create + list + delete is sufficient)
- Deployment configuration
- WebSocket or real-time features

## Technical Notes

### Workspace Addition

Add `"examples/todo-app"` to `Cargo.toml` workspace members:
```toml
members = ["autumn", "autumn-macros", "autumn-cli", "examples/hello", "examples/todo-app"]
```

### Database: Postgres

Use Postgres — the framework's target database. Include a `docker-compose.yml` for one-command setup:

```yaml
# examples/todo-app/docker-compose.yml
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_DB: todos
      POSTGRES_USER: autumn
      POSTGRES_PASSWORD: autumn
    ports:
      - "5432:5432"
```

The example's `Cargo.toml`:
```toml
[dependencies]
autumn = { path = "../../autumn" }
diesel = { version = "2", features = ["postgres"] }
serde = { version = "1", features = ["derive"] }
```

The example README should document: `docker compose up -d && cargo run -p todo-app`.

### Diesel Migration

```sql
-- migrations/00000000000000_create_todos/up.sql
CREATE TABLE todos (
    id SERIAL PRIMARY KEY,
    title TEXT NOT NULL,
    completed BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

-- migrations/00000000000000_create_todos/down.sql
DROP TABLE todos;
```

### Route Structure

**HTML routes (`routes/todos.rs`):**
- `GET /` → redirect to `/todos`
- `GET /todos` → list all todos (Maud template)
- `GET /todos/{id}` → single todo detail
- `POST /todos` → create from form, redirect to list
- `POST /todos/{id}/toggle` → toggle completed (htmx)
- `DELETE /todos/{id}` → delete (htmx)

**JSON API (`routes/api.rs`):**
- `GET /api/todos` → JSON array of todos
- `POST /api/todos` → create from JSON body

### Maud Templates

```rust
fn todo_list(todos: &[Todo]) -> Markup {
    html! {
        html {
            head {
                title { "Autumn Todo App" }
                link rel="stylesheet" href="/static/css/autumn.css" {}
                script src="/static/js/htmx.min.js" {}
            }
            body class="bg-gray-100 max-w-2xl mx-auto p-8" {
                h1 class="text-3xl font-bold mb-6" { "Todos" }
                // ... todo list with htmx delete buttons
                form method="POST" action="/todos" class="mt-4" {
                    input type="text" name="title" placeholder="New todo..."
                        class="border rounded px-3 py-2 mr-2" {}
                    button type="submit"
                        class="bg-blue-500 text-white px-4 py-2 rounded" {
                        "Add"
                    }
                }
            }
        }
    }
}
```

### Tailwind Strategy for Example

Use the real `build.rs` pipeline — the same one that `autumn new` generates. The example should demonstrate the production-grade approach, not a CDN shortcut. This means the example requires `autumn setup` (or a manually installed `tailwindcss` on PATH) to build CSS. The example README documents this prerequisite.

Include `build.rs`, `static/css/input.css`, and `tailwind.config.js` — identical to what `autumn new` produces. The htmx JS is served from the embedded `/static/js/htmx.min.js` route (S-022).

### autumn.toml for Example

```toml
[server]
port = 3000

[database]
url = "postgres://autumn:autumn@localhost:5432/todos"

[log]
level = "debug"
format = "Pretty"

[health]
path = "/health"
```

## Additional Notes

- The example uses Postgres + the real Tailwind build pipeline — exactly what `autumn new` generates. No shortcuts. If a developer reads this example and follows the same patterns, they get a production-grade setup.
- A `docker-compose.yml` makes Postgres one-command: `docker compose up -d`. This is the expected dev workflow.
- The example requires `autumn setup` (or `tailwindcss` on PATH) for the CSS build. Document this in the example README.
- The htmx integration should feel natural — delete a todo without page refresh, toggle completion inline. This is the "magic moment" that shows htmx + Maud working together.
