## User Story

I want **automated CI that tests Autumn on Linux, macOS, and Windows**
So that **cross-platform regressions are caught before merging and I can develop with confidence**

---

## Description

### Background

Autumn targets cross-platform usage (NFR-004). The framework uses platform-conditional code (`#[cfg(unix)]` for SIGTERM handling in S-030) and generates projects with platform-specific behavior (Windows `.exe` in `which()` fallback). Without CI, platform-specific bugs will slip through — especially since primary development happens on Windows.

This story creates the GitHub Actions workflow that runs on every push to `trunk` and every PR. It catches formatting drift, clippy warnings, test failures, and platform-specific compilation issues.

### Scope

**In scope:**
- `.github/workflows/ci.yml` workflow file
- **Lint job** (ubuntu-latest): `cargo fmt --all -- --check` + `cargo clippy --workspace --all-targets -- -D warnings`
- **Test job** (matrix: ubuntu, macOS, Windows): `cargo test --workspace`
- **MSRV job** (ubuntu-latest): `cargo check --workspace` with Rust 1.85.0
- `dtolnay/rust-toolchain` for Rust installation
- `Swatinem/rust-cache` for build artifact caching
- Trigger on push to `trunk` and PRs targeting `trunk`

**Out of scope:**
- Coverage reporting (can add later as separate job)
- Postgres service container (no DB integration tests yet — needed for S-037)
- Release/publish workflow (S-045)
- Badge in README (S-038)
- Benchmarking in CI
- Nightly Rust testing

## Technical Notes

### Workflow Structure

```yaml
name: CI

on:
  push:
    branches: [trunk]
  pull_request:
    branches: [trunk]

env:
  CARGO_TERM_COLOR: always
  RUST_BACKTRACE: 1

jobs:
  lint:
    name: Lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: rustfmt, clippy
      - uses: Swatinem/rust-cache@v2
      - name: Check formatting
        run: cargo fmt --all -- --check
      - name: Clippy
        run: cargo clippy --workspace --all-targets -- -D warnings

  test:
    name: Test (${{ matrix.os }})
    needs: lint
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - uses: Swatinem/rust-cache@v2
      - name: Run tests
        run: cargo test --workspace

  msrv:
    name: MSRV (1.85.0)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@1.85.0
      - uses: Swatinem/rust-cache@v2
      - name: Check MSRV compiles
        run: cargo check --workspace
```

### Design Decisions

**Two main jobs (lint + test):** Lint runs once on ubuntu (fast, no need for 3 OSes). Test matrix runs after lint passes. This gives fast feedback on formatting/lint failures without wasting matrix compute.

**MSRV as separate job:** Doesn't block the test matrix. Edition 2024 requires Rust 1.85.0 minimum. This job verifies we don't accidentally use features from newer Rust versions.

**`fail-fast: false`:** When a test fails on one OS, we still want to see results from the other OSes. This helps diagnose platform-specific vs universal failures.

**No `actions-rs`:** The `actions-rs/*` actions are unmaintained. `dtolnay/rust-toolchain` is actively maintained and simpler.

**No Postgres:** The `autumn` crate's DB tests verify config parsing, not actual connections. When S-037 adds E2E tests that need a database, we'll add a Postgres service container then.

### Platform Considerations

- **Windows:** `libsqlite3-sys` bundled build may be slow. `trybuild` compile-fail tests work but are slower. Path separators in template tests use `Path::join()` (already correct).
- **macOS:** `macos-latest` uses ARM (M-series) runners. Should work fine with stable Rust.
- **Linux:** Primary CI platform. Fastest runners.

## Additional Notes

- Caching with `Swatinem/rust-cache` typically reduces CI time by 50-70% on repeat runs. First run will be slow (full compilation).
- The MSRV of 1.85.0 matches the Rust edition 2024 stabilization. This is a conservative choice — we could pin to a newer version, but 1.85.0 ensures maximum compatibility.
- Future enhancements: coverage reporting (cargo-tarpaulin), dependency audit (cargo-deny), release automation. These are all post-v0.1 concerns.
- The workflow uses `CARGO_TERM_COLOR: always` for colored output in CI logs and `RUST_BACKTRACE: 1` for better error diagnostics.
