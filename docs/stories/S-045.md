## Description

### Background

This is the final story in the project — the one that puts Autumn on crates.io. Everything else is done: the framework works, the CLI scaffolds projects, the docs explain it, the example demonstrates it. This story handles the publication logistics: license files, metadata fields, dependency declarations, and the actual `cargo publish` commands.

Three crates must be published in dependency order: `autumn-macros` first (no deps on other workspace crates), then `autumn` (depends on `autumn-macros`), then `autumn-cli` (depends on `autumn`).

### Scope

**In scope:**
- Create LICENSE-MIT and LICENSE-APACHE files at workspace root
- Fix repository URL in workspace Cargo.toml (currently `markm`, should be `madmax983`)
- Add missing crates.io metadata: homepage, keywords, categories, readme
- Convert `autumn-macros` path dependency to version+path format for publishing
- Dry-run `cargo publish` on all three crates
- Actual `cargo publish` in dependency order
- Verify `cargo add autumn` works from a fresh project

**Out of scope:**
- Setting up a documentation site (docs.rs generates automatically from doc comments)
- GitHub releases or tags (can be done separately)
- Changelogs
- Post-publication marketing

## Technical Notes

### Publication Order

```
1. cargo publish -p autumn-macros    # no workspace deps
2. cargo publish -p autumn           # depends on autumn-macros
3. cargo publish -p autumn-cli       # depends on autumn (if applicable)
```

Wait ~30-60 seconds between publishes for crates.io indexing.

### Cargo.toml Changes

**Workspace `Cargo.toml`:**
```toml
[workspace.package]
repository = "https://github.com/madmax983/autumn"
homepage = "https://github.com/madmax983/autumn"
```

**`autumn/Cargo.toml`:**
```toml
keywords = ["web-framework", "axum", "async", "maud", "htmx"]
categories = ["web-programming", "web-programming::http-server", "asynchronous"]
readme = "../README.md"

[dependencies]
autumn-macros = { version = "0.1", path = "../autumn-macros" }
```

**`autumn-macros/Cargo.toml`:**
```toml
keywords = ["web-framework", "proc-macro", "axum"]
categories = ["web-programming"]
readme = "../README.md"
```

**`autumn-cli/Cargo.toml`:**
```toml
keywords = ["web-framework", "cli", "scaffolding", "tailwind"]
categories = ["command-line-utilities", "web-programming"]
readme = "../README.md"
```

### License Files

Standard MIT and Apache 2.0 license text. Copyright line:

```
Copyright 2026 Mark M (madmax983)
```

### Crate Name Availability

`autumn` may already be taken on crates.io. Fallback names:
- `autumn-web` (preferred fallback)
- `autumn-framework`

Check before publishing: `cargo search autumn | head -5`

### Pre-Publish Checklist Script

```bash
# Validate all crates
for crate in autumn-macros autumn autumn-cli; do
    echo "=== $crate ==="
    cargo publish --dry-run -p $crate 2>&1
    echo
done
```

### Post-Publication Verification

```bash
# From a fresh directory
cargo init test-autumn && cd test-autumn
cargo add autumn
# Should resolve and download from crates.io

cargo install autumn-cli
autumn --version
# Should print 0.1.0
```

## Additional Notes

- **Crate name conflict is the biggest risk.** If `autumn` is taken, the rename ripples through all documentation and the CLI. Check early.
- **Don't forget the git tag.** After publishing, create `v0.1.0` tag and push it: `git tag v0.1.0 && git push origin v0.1.0`.
- **docs.rs builds automatically** from the published crate. The doc comments (S-042) and feature flags (S-044) determine what shows up. Verify the docs.rs build succeeds after publishing.
- **Consider a GitHub release** alongside the tag — not required for this story but nice to have. Can include the README content and a link to docs.rs.
- The root README.md is already comprehensive (S-038). Using `readme = "../README.md"` in per-crate Cargo.toml makes it the crates.io landing page without duplication.
